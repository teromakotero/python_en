Introduction

Welcome to learn programming with Python. This course is divided into five parts: Statement, Loop, Condition, Variable and List. Each section has a theoretical part and challenges to test your skills and practice programming skills. Challenges include tasks that use Player Learning method. It is a challenging but effective way to learn programming.

Scratch materials are always running alongside this material. You can do all the tasks either just in Scratch, in the Python environment, or by combining the two creatively. So the tasks are quite or almost the same. Scratch tasks are named Scratch_to_Python_XX the corresponding Python task would be Python_from_Scratch_XX. So you can also study them at the same time.

You will first know what Player Learning means and how it helps you to learn programming skills effectively.

Idea of player learning comes from Harold Jarche.

Player Learning

Player Learning does not mean learning by playing a game, but the way a player can learn a new game in a new environment. The player tries to play the game without instructions and may fail many times. These failures are important and will help the player learn to play the game gradually. Making mistakes develops solutions. If a player gets stuck, he asks for help in online forums. Before or after, he may have already asked for help from a friend or a colleague. Once a player has found a solution, he can share it with others. This is very close to how many programmers have learned programming and how they work every day.

Here are the principles of Player Learning divided into seven points and how they will be implemented in the challenges of this course.

  1. Problem without solution
    • There can be little or no instructions at all in the challenges. They may just show the expected result.
    • The challenges measure not only programming skills, but also problem solving abilities.
    • Frustration and irritation in the fight for a difficult task are an integral part of the learning process.
    • There is no single right solution, but there may be several, one better than the other.
  2. The solution can be found without instructions
    • It's important to try different solutions and make the code better all the time.
  3. Errors teach
    • By trial and error, to insight.
    • Finding and solving errors causes in-depth learning.
  4. When making mistakes, solutions develop
    • Problem-solving abilities develop.
    • Solution models make it easier to solve new problems.
  5. Getting help from online forums
    • Programmers use online forums like Stack Overflow to help find solutions to programming problems.
  6. Help from friends
    • You can always ask for help from colleagues and other learners.
    • Nowadays, programming is very much a collaboration between experts.
  7. Sharing a solution with others
    • Many programmers share their code on GitHub and other similar sites.
    • The Scratch website at scratch.mit.edu has over 30 million projects distributed.
