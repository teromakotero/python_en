List Challenges

Python_from_Scratch_12

In this challenge you will learn how to make a shopping list with Scratch. The program asks: "What do you want?" And adds the item to the list. The program ends if you write "end".

In this challenge, we use the Player Learning method. You can come up with a solution through trials and mistakes. The method of trial and error is one of the basic methods for solving a problem.

Instructions

  1. Create the list "Shopping_list".
  2. Put the program to ask "What do you want?" (e.g., to a variable named shopping) and add user-entered purchases to the list until the user writes "end". After that, Pycat writes "Shopping List:" and the purchases added by the user to the list, and the program ends.
  3. You can use both while and for loops in your solution. With the for loop, you can easily get through the items in the "Shopping list" list for i in Shopping_list:).
  4. Pycat must be moved downwards to make the writing come in different lines.
  5. Finally, hide Pycat to see the list.
  6. Type below the text: # KIRJOITA TÄMÄN RIVIN ALAPUOLELLE OMA KOODISI
  7. Run the program with ►.

Python_from_Scratch_13

In this challenge you will learn how to debug, ie. find and correct errors in a code. The program should ask for a name and when you write "mix," it should mix the names in a random order. There are errors in the code that should be corrected.

In this challenge, we use the Player Learning method. The feeling of frustration and irritation is an important part of the learning process.

Instructions

  1. The names should be in random, not reverse, order when writing "mix".
  2. To mix you need the shuffle command with the list name.
  3. The program will continue to ask for the name even if it should end after mixing the names. How do you end it?

Python_from_Scratch_14

In this challenge, a program is created that sorts the words given by the user to their length lists. When the user writes "end", the program displays the sorted lists and stops asking the words.

In this challenge, we use the Player Learning method. The feeling of frustration and irritation is an important part of the learning process.

Instructions

  1. Make four lists: "2 or fewer letters", "3 letters", "4 letters" and "5 or more letters".
  2. Put the program to ask "Enter a word:" and place it in the "word" variable
  3. With the loop and conditionals the words are sorted into the right lists and when the word is "end", the list items are printed labeled: "2 or fewer letters", "3 letters", "4 letters" and "5 or more letters".
